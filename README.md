# 知っている範囲内でしか訳さないアプリ（プロトタイプ）

ローカルの単語帳（IndexedDB）に登録した語彙だけを使い、**未登録箇所を明示（ハイライト）**しながら「いまの自分の学習範囲」で訳すためのプロトタイプです。

## ディレクトリ構造

```text
known-only-translate/
|-- src/
|   |-- auth/                 # Supabase 認証コンテキスト
|   |-- components/           # 共通UIコンポーネント
|   |-- db/                   # IndexedDB/Dexie と型定義
|   |-- features/
|   |   |-- translate/        # 翻訳画面と翻訳サービス
|   |   `-- wordbook/         # 単語帳画面・編集・活用表
|   |-- grammar/              # 活用生成・アクセント・文生成ロジック
|   |-- lib/                  # ユーティリティ、Excel取込、同期補助
|   |-- pages/                # ルーティング単位のページ
|   |-- App.tsx
|   |-- App.css
|   `-- main.tsx
|-- import-template_B_*.tsv   # 一括登録用テンプレート
|-- import-template_B_README.md
|-- README.md
|-- package.json
|-- vercel.json
`-- dist/                     # ビルド生成物
```

## ねらい（コンセプト）

- **単語帳**: 人が登録するのは最小限（見出し語・品詞・意味・活用タイプ）
- **文法エンジン（試作）**: 文のときは単語の置換だけではなく、活用タイプとルールで文を生成
  - 例: `Αγοράζω μήλο.` → `私は りんごを 買う。`

## できること

- **単語モード**: 単語帳に登録済みの語彙だけで置換し、未登録箇所をハイライト
  - 動詞/名詞は、見出し語＋活用タイプから生成できる範囲の活用形でも照合します（例: `αγοράζεις` など）
- 形容詞/疑問詞（最小対応）は、見出し語から生成できる範囲の活用形でも照合します
- **文モード（試作）**: ギリシャ語→日本語で、最小ルールにより省略主語/助詞を補って文を生成
  - 動詞（`-ω / -εις / -ει / -ουμε / -ετε / -ουν(ε)`）→ `私は / あなたは / 彼/彼女は / 私たちは / あなたたちは / 彼ら/彼女らは`
  - 名詞は簡易に格を推定し、`を / が` を補う（未確定な場合は位置ヒューリスティック）
- **単語帳**:
  - メモ、例文、関連語、タグ（複数可）
    - 例文は **「原文＋訳」** のペアで保存できます（詳細画面では原文/訳を表示）
    - 文モードでは、入力文が例文（原文）と **完全一致**し、かつ訳が登録されている場合は、その訳を最優先で返します（最小実装）
  - 活用（マトリックス）表示＋編集（上書きした形は単語モード/文モードの照合にも反映）
  - **名詞**の活用表では、**語形が入っているセルだけ**冠詞（`ο/η/το` 等）を付けます（セルが空のときに冠詞だけが残るのを防ぎます。複数形のみの語で単数を空にする運用に向きます）
  - **名詞**で自動推定できない語尾（例: `-ούς` の `παππούς`）でも、編集画面で活用を手入力（`n_nom_sg` 等）すれば詳細に活用表を表示します
  - **名詞（男性/女性/中性）**（内部: `tri_gender`）: 数詞など、性ごとに形が異なる名詞を、形容詞と同様の **18マス**（単数・複数 × 男・女・中 × ～は/～の/～を）で登録・編集できます。手入力した語形は単語モード/文モードの照合にも反映されます
  - 一覧フィルター（モーダル）: **品詞 / タグ / アルファベット（ギリシャ文字 Α〜Ω、複数選択可）**
    - 品詞で **名詞** を選ぶと **性別**（男性 / 女性 / 男性・女性 / 中性 / **男性・女性・中性**）を追加で絞り込み（複数選択は OR）
    - 品詞で **動詞** を選ぶと **タイプ**（Α / Β1 / Β2 / ΑΒ / Γ1 / Γ2）を追加で絞り込み（一覧バッジと同じ分類・複数選択は OR）
    - アルファベットの「#」は Α〜Ω 以外（数字・記号など）。トノスは無視して判定
  - 変化する語尾を赤字で強調表示（動詞/名詞/形容詞）
  - 単語削除は確認モーダルを表示（画面中央）
  - **関連語**: チップをタップすると該当項目へジャンプ。**単語帳に無い語を選んでもメッセージは出ません**（そのまま詳細を見続けられます）
  - スマホでは単語詳細で「編集」を開いたとき、画面先頭へスクロールして入力欄が見えやすくなります
- 名詞（男性/女性/中性・試作）:
  - 性で **「男性/女性/中性」** を選ぶと、登録画面・単語帳編集で **18マスの活用表** が開きます（自動推定はしません。すべて手入力）
  - 上書きキーは `n_m_nom_sg` … `n_n_acc_pl`（`n_m_` = 男性、`n_f_` = 女性、`n_n_` = 中性）
  - 例（数詞「1」）: 見出し `ένας`、男性 単数 ～は `ένας` / 女性 `μία` / 中性 `ένα` など
  - 詳細画面では男・女・中の3列マトリックス（冠詞付き）を表示します
- 形容詞/疑問詞（試作）:
  - 男性形（例: `μεγάλος`）を登録すると、女/中（＋複数・格）をマトリックス表示（最小対応）
  - **疑問詞**のうち、形容詞型の活用表を出すのは **`ποιός` / `πόσος` の見出しのみ**（それ以外の疑問詞は活用ブロックを表示しません）
  - `πολύς` のような不規則も最小対応しています
- **人称代名詞**:
  - 共通の標準表（10行×5列: 一単〜三中複 × ～は／～の強・弱／～を強・弱）を表示・編集できます。登録時は表の語形が照合にも使われます
- **認証・ルーティング**（Supabase 設定時）:
  - 初回アクセスは **ログイン画面**（`/login`）。ログイン / 新規登録後は **単語帳**（`/wordbook`）へ
  - ヘッダーにメールアドレス表示と **ログアウト**（アイコン）
  - Supabase 未設定時はログイン画面から **単語帳へ（ローカルモード）** で利用可能（認証なし）
- **設定**:
  - 設定トップに **表示言語 / タグ設定 / 一括登録 / クラウド同期** の4項目
  - **表示言語**: 日本語 / English / Ελληνικά を選択可能（**保存できるのは日本語のみ**。それ以外で保存すると「日本語以外の表示言語は今後対応予定です。」と表示）
  - タグ設定: タグ追加、プルダウンで選択して削除（単語側のタグは残ります）
  - **クラウド同期**（Supabase）: 同期状態の表示、手動のクラウド保存 / クラウドから復元、競合時の選択（ログインは `/login`）
  - 一括登録（Excel）: テンプレをダウンロードして `.xlsx` をアップロード（後述）。`word` のタグ列に**まだタグ設定に無い名前**があれば、**自動でタグ設定へ追加**し、画面上で通知します
- **重複チェック**:
  - 既に登録済みのタグ/単語を追加しようとすると、モーダルでメッセージを表示します
- **見出し語（ギリシャ語）の表記**:
  - 保存時は **Unicode NFC と前後空白の整理**のみ行い、**大文字・小文字は入力どおり**保持します（曜日・月など先頭大文字の語も登録・編集できます）
  - 翻訳の照合・重複判定・活用推定は **`normalizeToken`（小文字化）** で行うため、`Δευτέρα` と `δευτέρα` は同一語としてマッチします

- `src/auth/`: 認証コンテキスト（Supabase セッション）

## コード構成（文法/活用まわり）

このプロトタイプでは、活用生成やアクセント処理を `src/grammar/` に寄せています（UIは `src/features/` 側）。

- **`src/grammar/accent.ts`**
  - トノスの除去/付与、アクセント位置推定
  - **母音ユニット**（例: `ου/ευ/αι/ει/οι/αυ/ηυ` など）を 1 つとして扱う API も持ちます
    - 例: `addTonosOnNthFromEndVowelUnit(...)`
    - 例: `applyAccentFromByVowelUnit(...)`
  - 二重母音は通常 **後ろ側**の母音にトノス（例: `νερό` → `νερού`）。`ια/ιε/ιο` は **前側**にもあり得ます（例: `τραπεζαρία` の `ί`）
  - 母音ユニットで検出できない場合は、文字単位の `applyAccentFrom(...)` にフォールバックします
- **`src/grammar/verb.ts`**
  - 動詞の最小活用（現在/過去/未来/να/命令、アオリスト）を生成
  - UIラベルは持たず、`person: '1sg' | ...` のような **純データ**を返します
- **`src/grammar/noun.ts`**
  - 名詞の自動語形（`nounAutoForms`）とマトリックス用データ（`nounMatrix`）を生成
  - `endingsByCell`（語尾表）や `resolveNounTypeForMatrix`（名詞タイプ決定）もここに集約
  - どちらも `applyAccentFromByVowelUnit` でトノスを付与します（表示用 `nounMatrix` と照合用 `nounAutoForms` の挙動を揃えています）
  - `nounMatrix` は **冠詞なし・UIラベルなし**の語形データを返し、冠詞表示（`ο/η/το` 等）や「単数/複数」「～は/～の/～を」は UI 側（`InflectionSection.tsx`）で組み立てます（**空セルには冠詞を付けない**のも UI 側）
- **`src/grammar/nounTriGender.ts`**
  - 名詞（`tri_gender`）の手動上書きキー一覧（`NOUN_TRI_GENDER_KEYS`）と、照合用の語形収集（`collectNounTriGenderForms` 等）
- **`src/features/wordbook/components/NounTriGenderOverridesEditor.tsx`**
  - 登録画面・単語帳編集の 18マス入力 UI（男性/女性/中性 × 単複 × 格）
- **`src/grammar/adjective.ts`**
  - 形容詞/疑問詞の最小活用（マトリックス/照合用の語形集合）を生成
- **`src/grammar/personalPronoun.ts`**
  - 人称代名詞の固定活用表と、単語モード照合用の語形集合
- **`src/grammar/engine.ts`**
  - 文モードのコア（純ロジック）: `translateSentenceForeignToJaCore(...)`
- **`src/features/translate/services/sentenceTranslate.ts`**
  - 文モードのI/O（DB読み/例文完全一致の優先など）: `translateSentenceForeignToJa(...)`
- **`src/features/translate/services/knownOnlyTranslate.ts`**
  - 単語モード翻訳: `translateKnownOnly(...)`（見出し/別形の検索は `entryForeignLookup.ts` で **小文字化キー**により大文字小文字を区別しません）

## 使い方（最短）

1. 起動後、**ログイン画面**（Supabase 設定時はアカウント作成またはログイン / 未設定時は「単語帳へ（ローカルモード）」）→ **単語帳** が開きます（`/` は `/wordbook` へリダイレクト）
2. `登録` 画面の **「登録」** ボタンで単語を追加
   - 品詞: **名詞 / 動詞 / 形容詞 / 副詞 / 人称代名詞 / 疑問詞 / 前置詞 / 接続詞 / その他**
   - **見出し語**は大文字・小文字をそのまま保存します（照合は小文字化で行うため、文中の表記ゆれでもヒットします）
   - **動詞**は活用タイプを選択（例：`Α : -νω/-ζω` など）
     - 活用タイプが **その他** の場合は、活用は編集画面で手入力してください（活用表も手入力で表示します）
   - **名詞**は性を選択
     - **男性 / 女性 / 中性 / 男性・女性**（通性）: 活用タイプは見出し語の語尾から自動推定（単語帳で上書きも可）
     - **男性/女性/中性**: 18マスの活用表に手入力（数詞など。1つ以上のマスが埋まっていれば登録可）
   - （任意）タグ/例文/関連語/メモを追加
3. `翻訳` で **「変換」** を押して
   - 単語単位なら **モード=単語**
   - 文なら（外国語→日本語で） **モード=文（試作）**
4. `単語帳` で詳細確認・編集（関連語はタップでジャンプ・未登録語はメッセージなし）
5. `設定` で
   - `表示言語`（選択肢: 日本語 / English / Ελληνικά。UI の切り替えは未実装で、**日本語のみ保存可**）
   - `タグ設定`（タグの追加/削除）
   - `一括登録`（Excelのアップロードでまとめて登録）
   - `クラウド同期`（手動保存・復元、同期状態。ログインは `/login`）

## データ保存とバックアップについて

- **保存先**: ブラウザの IndexedDB（DB名: `knownOnlyTranslate`）
- **スキーマ**: Dexie のバージョンでマイグレーションします。見出し語の **ケース保持**はアプリ側の保存ロジックとマイグレーション（v12 以降）で揃えています
- **設定（settings）**: `uiLanguage`（`ja` / `en` / `el`）と `tags` を保持。表示言語の UI 切り替えは未実装（`ja` のみ保存）
- **クラウド同期のペイロード**: `src/lib/backupPayload.ts` が単語帳＋設定を1つの JSON にまとめます（Supabase の `user_state.data` に保存）
  - 形式の目安:
    - `kind: "known-only-translate-backup"`
    - `version: 1`
    - `settings` / `entries`
  - 設定画面からの JSON エクスポート/インポート UI はありません（クラウド同期で端末間共有）

## 一括登録（Excel）

`設定 > 一括登録` から、テンプレExcelを使ってまとめて登録できます。

- **テンプレ**: 画面上の「テンプレートをダウンロード」から取得できます（`template.xlsx`）
- **必須シート**: `word`, `noun`, `adjective`, `verb`
- **必須列（wordシート）**: `foreignLemma`, `pos`, `meaningJa`
- **品詞ごとの必須行**
  - `pos=noun` の行は、`noun` シートに同じ `foreignLemma` の行が無いとエラー
  - `pos=verb` の行は、`verb` シートに同じ `foreignLemma` の行が無いとエラー
  - `pos=adjective` / `pronoun_interrogative` の行は、`adjective` シートに同じ `foreignLemma` の行が無いとエラー
- **エラー表示**: ダイアログに `word` の「何行目の何が不足か」を一覧表示します
- **重複単語**: 既存エントリと **正規化（小文字化）した見出しが同じ**場合に重複とみなします（大文字小文字だけ違う表記は重複扱い）。ダイアログ **「重複単語が見つかりました」** で **上書き / スキップ / キャンセル** を選べます（タグ自動追加があった場合は同じダイアログ内に説明が出ます）
- **活用の自動生成（空欄補完）**
  - `noun` / `adjective` / `verb` シートの活用欄が空欄の場合、見出し語とタイプから生成できる範囲で自動生成し、照合用の `foreignForms` に反映します
  - 動詞の `inflectionType=none（その他）` は自動生成しません（手入力前提）
  - 名詞の `nounGender=tri_gender`（男性/女性/中性）は自動生成しません。`noun` シートに `n_m_nom_sg` などの列を追加して手入力してください（`pickOverrides` が `n_` 始まりの列を読み込みます）

## 公開（Vercel）とクラウド同期（Supabase）

### 公開（Vercel）

1. GitHubにpush
2. VercelでGitHubリポジトリをImportしてDeploy（Viteなので通常は自動設定でOK）
   - Build: `npm run build`
   - Output: `dist`
   - ルート直下の `vercel.json` で、すべてのパスを `index.html` にフォールバックします（`/wordbook` などを再読み込みしても 404 にならない）

### Vercelの環境変数（Supabase）

Vercelのプロジェクトで環境変数を設定し、設定後にRedeployします。

- `VITE_SUPABASE_URL`: Supabaseの Project URL（例: `https://<ref>.supabase.co`）
- `VITE_SUPABASE_ANON_KEY`: Supabaseの `anon public` key

### Supabase側の準備（メール+パスワード + 1ユーザー=1単語帳）

- Authenticationで **Email（メール/パスワード）** を有効化
- DBに `user_state` テーブル（バックアップJSONを丸ごと保存）を作成し、RLSで本人だけアクセスできるようにします

```sql
create table if not exists public.user_state (
  user_id uuid primary key references auth.users (id) on delete cascade,
  data jsonb not null,
  updated_at timestamptz not null default now()
);

alter table public.user_state enable row level security;

create policy "user_state_select_own"
on public.user_state for select
using (user_id = auth.uid());

create policy "user_state_insert_own"
on public.user_state for insert
with check (user_id = auth.uid());

create policy "user_state_update_own"
on public.user_state for update
using (user_id = auth.uid())
with check (user_id = auth.uid());

create policy "user_state_delete_own"
on public.user_state for delete
using (user_id = auth.uid());
```

### ローカルのデータを公開URLへ移す

方法は2つあります。

- **クラウド同期（おすすめ）**:
  - Supabaseが設定され、かつログイン済みの場合は、単語帳の変更が自動でクラウドへ保存されます
  - 初回は「ローカルが空で、クラウドにデータがある」場合に自動で復元されます
  - 手動操作は `設定 > クラウド同期`（クラウドへ保存 / クラウドから復元）

### PCとスマホで同じデータを使う

- PC/スマホそれぞれで **公開URL** を開き、同じアカウントでログインします。
- ログイン済みであれば、単語の追加/編集/削除などの変更は数秒以内にクラウドへ自動保存され、もう一方の端末にも反映されます。

### クラウド同期と競合（データの食い違い）

同じ Supabase アカウントで **Chrome・Firefox・スマホ** などにログインすると、単語帳を共有できます。変更は数秒以内に自動でクラウドへ保存されます（`設定 > クラウド同期` で手動操作も可能）。

**競合が起きるのは次のようなときです。**

1. **2つ以上の端末・ブラウザ**で、同じアカウントにログインしている
2. **両方で単語を編集**した（一方がオフラインのまま編集を続けた場合も含む）
3. クラウド上の「正本」が、どちらの内容か曖昧になった

**このアプリの扱い（試作）:**

| 状況 | 自動同期の動き |
|---|---|
| この端末に未保存の変更がある | クラウドからの上書きはしない。こちらをクラウドへ送る |
| この端末に変更がない | クラウドの内容を取り込む |
| ローカルが空でクラウドにデータがある | 起動時などにクラウドから自動復元 |

- クラウドの正本は **単語帳JSONを丸ごと1つ**保存する方式です。**単語ごとのマージはしません**。
- **最後にクラウドへ保存した内容**が残ります（別端末の変更は上書きされます）。
- 「クラウドから復元」で、未同期のローカル変更があるときは **競合ダイアログ**で「この端末を送る / クラウドで上書き」を選べます。

注意:
- Supabase 未設定（`.env` なし）のビルドでは、クラウド同期とログイン必須は無効です（ローカル IndexedDB のみ）。

## 動詞活用タイプ（試作）

現状、動詞は以下の活用タイプを選べます（増加中）。

- **`その他`**（内部: `none`）: 活用表は手入力（上書き）で運用
- **`Α : -νω/-ζω`**（内部: `verb_pres_act_-ω`）
- **`Α : -γω/-χω/-χνω`**（内部: `verb_pres_act_-γω_-χω_-χνω`。このタイプを選んだ `-ζω` も ξ 系）
- **`Α : -πω/-φω/-βω/-εύω`**（内部: `verb_pres_act_-πω_-φω_-βω_-εύω`）
- **`ΑΒ : -ω（αόρ. 不規則）`**（内部: `verb_pres_act_AB`）
- **`Β1 : -άω（αόρ. -ησα）`**（内部: `verb_pres_act_B1_-άω_-ησα`）
- **`Β1 : -άω（αόρ. -εσα）`**（内部: `verb_pres_act_B1_-άω_-εσα`）
- **`Β1 : -άω（αόρ. -ασα）`**（内部: `verb_pres_act_B1_-άω_-ασα`）
- **`Β2 : -ώ（αόρ. -ησα）`**（内部: `verb_pres_act_B2_-ώ_-ησα`）
- **`Β2 : -ώ（αόρ. -ασα）`**（内部: `verb_pres_act_B2_-ώ_-ασα`）
- **`Β2 : -ώ（αόρ. -εσα）`**（内部: `verb_pres_act_B2_-ώ_-εσα`）
- **`Γ1 : -ομαι`**（内部: `verb_pres_mid_Γ1_-ομαι`）
- **`Γ2 : -άμαι`**（内部: `verb_pres_mid_Γ2_-άμαι`）

### Γ（-ομαι / -άμαι）の最小ルール（表示・照合向け）

例:

- `εργάζομαι → εργάστηκα`（Γ1）
- `κοιμάμαι → κοιμήθηκα`（Γ2）

対応範囲（最小）:

- **現在（継続・繰返し）**
  - Γ1: `-ομαι/-εσαι/-εται/-όμαστε/-εστε/-ονται`
  - Γ2: `-άμαι/-άσαι/-άται/-όμαστε/-άστε/-όνται`
- **過去（継続・繰返し）**（未完了過去・最小）: `-όμουν/-όσουν/-όταν/-όμασταν/-όσασταν/-όνταν`
- **未来（継続・繰返し）**: `θα + 現在`（表示用）
- **アオリスト過去**（最小）
  - Γ1: `-στηκα` 系（例: `εργάζομαι → εργάστηκα`。`...ζ` は `ζ→σ` を入れて生成）
  - Γ2: `-ήθηκα` 系（例: `κοιμάμαι → κοιμήθηκα`）
- **アオリスト未来**（最小）
  - Γ1: `θα εργαστώ` 系
  - Γ2: `θα κοιμηθώ` 系
- **命令（2人称のみ・最小）**
  - 継続・繰返し: 2複のみ（Γ1: `...εστε` / Γ2: `...αστε`。トノスは後述のルールで付与）
  - アオリスト: 2単/2複（例: `εργάσου/εργαστείτε`, `κοιμήσου/κοιμηθείτε`）

### 活用マトリックスの列

動詞のマトリックスは、**「να」の右に命令形（命令）**を表示します。

- **命令列**は **2人称（2単/2複）のみ**が対象で、それ以外（1人称/3人称）は空欄になります。

### Β1（-άω）の最小ルール（表示・照合向け）

例:

- `απαντάω / βοηθάω / ζητάω`（-ησα 型）
- `γελάω / πεινάω / χαλάω`（-ασα 型）

対応範囲（最小）:

- **現在（継続・繰返し）**: `-άω/-άς/-άει/-άμε/-άτε/-άνε`（`απαντά` も 3単として許容）
- **過去（継続・繰返し）**: `-ούσα` 系（例: `απαντούσα`）
- **アオリスト過去**:
  - `-ησα` 型: `απαντάω → απάντησα`
  - `-εσα` 型: `πονάω → πόνεσα` / `φοράω → φόρεσα`
  - `-ασα` 型: `γελάω → γέλασα`
- **命令（2人称のみ）**
  - Imperfective: 2単 `-α`（例: `απάντα`）、2複 `-ατε`（トノスは後述のルールで付与）
  - Perfective: 2単 `-ησε / -ασε`（例: `απάντησε / γέλασε`）、2複 `-ήστε / -άστε`（例: `απαντήστε / γελάστε`）

### Β2（-ώ）の最小ルール（表示・照合向け）

例:

- `δημιουργώ → δημιούργησα`（-ησα 型）
- `εξηγώ → εξήγησα`（-ησα 型）

対応範囲（最小）:

- **現在（継続・繰返し）**: `-ώ/-είς/-εί/-ούμε/-είτε/-ούν`
- **過去（継続・繰返し）**: `-ούσα` 系（例: `δημιουργούσα`）
- **アオリスト過去**:
  - `-ησα` 型: 語幹 + `ησα`（例: `δημιουργώ → δημιούργησα`）
  - `-ασα` 型: 語幹 + `ασα`（最小ルール）
  - `-εσα` 型: 語幹 + `εσα`（最小ルール）
- **命令（2人称のみ）**
  - Imperfective: 2単 `-εί`、2複 `-ειτε`（トノスは後述のルールで付与）
  - Perfective: 2単 `-σε`、2複 `-στε`（アオリスト語幹から生成）

### 命令形のトノス（アクセント）ルール（試作）

命令形のトノス位置は「母音ユニット数」で決めます。

- **母音ユニット**: `εύ/ου/αι/ει/οι/αυ/ηυ` を **1つ**として数えます（例: `δουλεύω` は `ου + εύ + ω` で 3）。
- **2単（命令）**:
  - 母音ユニット数が **2以下** → 後ろから **2番目**
  - それ以外 → 後ろから **3番目**
- **2複（命令）**:
  - 継続・繰返し → 後ろから **3番目**
  - アオリスト → 後ろから **2番目**

### アオリスト（ψ/ξ 系）の最小ルール

活用タイプによっては、語幹変化を最小ルールで扱います。

- `-γω/-χω/-χνω`（このタイプを選んだ `-ζω` を含む） → **ξ 系**（例: `τρέχω → έτρεξα`, `πειράζω → πείραξα`）
- `-πω/-φω/-βω/-εύω` → **ψ 系**（例: `γράφω → έγραψα`, `χορεύω → χόρεψα`）

### 語尾ハイライト（赤/青）

単語帳の活用マトリックスでは、**人称語尾を赤**、（アオリスト等の）**目印を青**で強調します。

- **赤（語尾）**: 人称語尾（例: `αγόρασα` の `α`、`θα αγοράσω` の `ω`）
- **青（目印）**:
  - アオリストの `σ/ψ/ξ`（例: `αγόρα` + `σ` + `α`）
  - Β1（-ησα）の `ησ`（例: `απάντ` + `ησ` + `α`）
  - Γ（中動態・最小）の `στ/ηθ`（例: `εργα` + `στ` + `ηκα`、`κοιμ` + `ηθ` + `ηκα`）
  - Β1/Β2 未完了過去の `ούσ`（例: `απαντ` + `ούσ` + `α` / `δημιουργ` + `ούσ` + `α`）
  - ΑΒ 未完了過去/命令（2単）の `γ`（例: `έκαι` + `γ` + `α`、`καί` + `γ` + `ε`）

※ どれも厳密なギリシャ語活用の完全実装ではなく、単語帳の表示・照合を目的にしたヒューリスティックです。例外は「活用の上書き」で対応できます。

### 名詞のトノス（アクセント）ルール（試作）

名詞の活用表・照合では、見出し語のアクセント位置を各語形へ写します（`applyAccentFromByVowelUnit`）。

- **中性 `-ο` 型**（内部: `noun_2nd_neut_-ο`、例: `νερό`）
  - 語幹 + `ο/α/ου/ων` を生成し、見出しのアクセント位置を写す（例: `νερό`, `νερού`, `νερά`, `νερών`）
- **女性 `-α` 型**（内部: `noun_fem_-α` / `noun_fem_-ά`、例: `τραπεζαρία`）
  - 語尾 `-α` で推定（`-ά` は見出しが最後音節アクセントのとき）
  - **`-ία` 語尾**（例: `τραπεζαρία`）: 語末の `ια` で `ί` にトノスがある場合を検出し、活用形へ写す
  - **属格単 `-ίας`**: `ια` を分離し、`-άς` の `ά` にトノス（例: `τραπεζαριάς`）
  - **複数 `-ιες`**: `ιε` は 1 ユニットのまま（例: `τραπεζαρίες`）
  - **属格複 `-ιών`**: 語末 `ων` にトノス（例: `τραπεζαριών`）

母音ユニットの数え方（名詞・動詞共通の最小ルール）:

- `ου/ευ/αι/ει/οι/αυ/ηυ` などは通常 **1 ユニット**（トノスは後ろ母音が既定）
- `ια/ιε/ιο` も 1 ユニットとして扱うが、トノスは **前後どちらにも**あり得る
- `-ίας` の `ια` だけは **分離**（`ς` の直前の `α` が属格アクセントになるため）

※ 上記もヒューリスティックです。不規則なアクセント移動や例外は「活用の上書き」で対応してください。
